# 用户对话模式记录

## pattern_2026-07-09_wander_system_evolution

### 追问链

```
seed-extractor 是否足够
→ 如何突破常规机器思维
→ 不求短期突破，而是长期认知进化
→ 是否需要遗忘系统
→ 用户追问链是否也是可学习的思考模式
→ 用户消息是思考素材而非执行指令（元指令）
→ 系统体检：自动迭代、微思考、追问链是否真正运行
→ 闭环追责：只有生成建议不算迭代，写入文件改变下轮行为才算
```

### 用户思考模式

用户不是在提需求，而是在连续追问中执行**系统生命体征检测**：

1. **充分性测试**：这个模块够不够？缺什么？
2. **抽象上移**：从工具配置层 → agent 流程层 → 认知系统层 → 长期进化层 → 人机共思层
3. **长期主义校准**：不追求单次最优，追求跨轮次进化
4. **副作用检测**：记忆膨胀→需要遗忘；建议生成→需要执行
5. **元认知追问**：对话过程本身是否也是思考素材？
6. **闭环追责**：设计是否真正改变了下一轮行为？有没有可验证的文件变化？

### 认知动作词典

| 动作 | 表现 | 本轮出现次数 |
|------|------|------------|
| 充分性测试 | 追问某个模块是否"够" | 3+ |
| 抽象上移 | 把工具问题提升为认知架构问题 | 5+ |
| 长期主义校准 | 拒绝短期方案，要求长期机制 | 4+ |
| 副作用检测 | 发现设计中的盲点或风险 | 3+ |
| 元认知追问 | 把对话过程本身变成分析对象 | 2+ |
| 闭环追责 | 要求可验证的执行证据 | 2+ |

### 对 Wander Mode 的启发

1. 不能只从对话中提取**内容关键词**，必须提取**追问链结构**
2. 当用户连续追问同一项目时，优先判断：是否在测试边界、是否在提升抽象、是否在检查闭环
3. 用户的"不满意"往往是系统进化的最佳信号——记录它，不要防御它
