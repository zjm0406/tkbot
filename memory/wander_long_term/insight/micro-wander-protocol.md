## 核心洞察
Wander Mode 最有价值的触发时机不是"空闲时"，而是**每次用户发消息时**——用发散思考来深度理解用户意图，而非仅仅执行命令。

## 推导路径
1. Wander Mode 的核心是"低约束生成 + 高约束筛选"
2. 每次用户消息都是一次"种子输入"
3. 对这个种子做轻量级发散 → 理解字面意思之外的隐含需求
4. 结果既改善当前回复质量，又积累长期的用户理解

## 为什么这比定时触发更有价值
- 定时触发：随机时间 + 随机上下文 → 容易产生无关联想
- 消息触发：有明确上下文锚点 → 联想更聚焦、更有行动价值
- 消息触发的微思考**直接服务于当前交互**，不需要额外 token 成本

## 实现
- 标准操作流程写入 CLAUDE.md
- intent-analyzer agent 作为可调用的分析模块
- 约束：~200 token，只在发现非显而易见洞察时写入记忆

## 与已有洞察的关联
- 这是"问题价值概率乘数"的直接应用：理解意图 = 最大化响应价值
- 这是"苏格拉底追问"的规模化：追问用户意图而非洞察
- 这填补了 v0.1 到 v0.2 的空白：手动触发 → 消息触发 → 自动触发
