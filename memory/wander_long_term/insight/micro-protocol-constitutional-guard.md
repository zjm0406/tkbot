# 微思考协议宪法守卫

## 核心规则（合题）

> **内容层可以影子进化，认证层必须外部审查。**

## 双层架构

```
普通层 (Ordinary) — 步骤 1-6
  影响"怎么回答"
  shadow_applied → 3次对话验证 → committed

认证层 (Constitutional) — 步骤 7 + 变更表 + 双层定义
  影响"什么能改协议本身"
  只能 proposal，禁止自动应用
  需要独立验证（下一场对决 或 用户确认）
```

## 六个受保护对象

以下任何修改 = 认证层改动 = proposal-only：
1. 步骤总数增减
2. 步骤 7 权限级别
3. shadow_applied → committed 晋升条件
4. TTL 和验证条件定义
5. 双层架构定义
6. 认证层边界

## 对决验证

对决#003 ClaudeCode 48 vs Codex 46 = 平局。双方独立收敛到"宪法层 ≠ 普通层"——一方叫受保护对象，一方叫默认拒绝宪法。这是结构必然性，不是偏好。
