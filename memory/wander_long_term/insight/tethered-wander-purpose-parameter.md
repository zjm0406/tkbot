## 核心洞察
Wander Mode 不应只有"纯随机发散"和"完全定向研究"两个极端。中间态——**有锚点的发散**——才是最有生产力的模式：1-2 个种子沿目的方向提取，其余保持自由来源，目的种子和随机种子之间的张力本身会产生最有价值的联想。

## 推导路径
用户反馈"是不是每次思考时可以增加一个目的性参数" → 意识到当前设计缺失了"探索-利用"的中间调节参数 → 类比科学研究中的 serendipity（有准备的头脑在随机中发现有意义的东西）→ 设计 `--purpose` + `--purpose-weight` 参数

## 实现
- `--purpose "目标"`：为本次思考设定方向锚点
- `--purpose-weight 低|中|高`：控制目的性强度（1/2/3 个种子定向）
- 目的种子和随机种子之间的**语义张力**会产生最有价值的联想——这不是 bug，是 feature

## 与已有洞察的关联
- 与"熵增-压缩平衡"直接相关：目的是压缩方向，随机是熵增方向，purpose-weight 调节二者的比值
- 与"问题价值概率乘数"相关：有目的的思考 = 更高的上下文相关性 = 更高的行动概率
- 与"涌现事后可解释"相关：事后回看，目的种子和随机种子之间产生的连接往往看起来"必然"，但事前不可预测
