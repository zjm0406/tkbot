# /wander — 空闲发散思考模式

触发智能体进行一次**有限发散思考回合**。

**核心原则（来自 v0.1 实践验证）**：
- 交替发散与收敛，而非先全部发散再全部收敛（熵增-压缩平衡，24分）
- 每个阶段都**直接向用户展示**中间结果，不藏到后台 agent 或文件里
- 每个阶段输出必须先压缩（一句话摘要）再传递给下一阶段（认知架构=信息压缩，21分）
- 好的问题比好的答案更有长期价值（问题价值概率乘数，24分）

## 参数
- `--seeds "词1,词2,词3"`：手动指定种子词，跳过自动提取
- `--no-save`：不写入记忆，只展示思考结果

## 执行流程（全部在主对话中逐步展示）

### 阶段 0：准备
1. 读取 `memory/MEMORY.md` 了解已有记忆和用户偏好
2. 读取 `data/word_pool.json` 加载随机词库
3. 告知用户："开始 Wander Mode 发散思考…"

### 阶段 1：种子提取（发散）→ 立即展示
使用 general-purpose agent 调用 seed-extractor 逻辑，提取 3-5 个种子。

**展示格式**：
```
## 🌱 种子提取

| # | 种子 | 来源 | 优先级权重 |
|---|------|------|-----------|
| 1 | xxx | 最近对话 | 高（最新上下文）|
| 2 | xxx | 长期记忆 | 高（用户持续关注）|
| 3 | xxx | 开放问题 | 中（未解决）|
| 4 | xxx | 随机词库 | 低（偶然性）|

> 选择理由：xxx
```

**记忆层级优先级**：wander_insight > wander_open_question > wander_buffer > 随机词库
（来自"认知记忆分层映射到种子提取"，18分）

### 阶段 2：发散联想（发散）→ 逐条展示
对每个种子生成三层联想链。**每个种子展示一条完整的联想链**。

```
## 🔗 种子 N：xxx

L1 直接 → xxx、xxx、xxx
L2 间接 → xxx、xxx、xxx
L3 远距 → xxx、xxx、xxx

> 🎯 链摘要：[一句话核心洞察]
```

**约束**：每层必须向新语义方向扩展，禁止回环。只有 3 层。

### 阶段 3：中间收敛（收敛）→ 展示并决定是否继续
对所有联想链做一次快速预筛选：
- 检查是否出现语义回环（后续层出现了前层已覆盖的概念）
- 检查联想链之间是否有明显重复
- 标记有潜力的联想方向

```
## 🎯 中间收敛

| 种子 | 链质量 | 是否有回环 | 是否重复 | 继续？ |
|------|--------|-----------|---------|--------|
| xxx | 高 | 否 | 否 | ✅ |
| xxx | 中 | 否 | 是→合并 | ✅ |
| xxx | 低 | 是 | 否 | ⚠️ 降权 |

> 本次联想共 X 条有效链，进入深度评分阶段
```

这是"熵增-压缩平衡"的具体实现——发散之后必须有一次压缩再进入下一步。

### 阶段 4：价值筛选（收敛）→ 展示完整评分表
对所有联想结果进行五维评分。

```
## 📊 五维评分

| # | 内容摘要 | N | R | A | C | F | 总分 | 决策 |
|---|---------|---|---|---|---|---|------|------|
| 1 | xxx | 5 | 4 | 3 | 5 | 4 | 21 | 🔵 长期 |
| 2 | xxx | 3 | 3 | 2 | 4 | 3 | 15 | 🟡 缓冲 |
| 3 | xxx | 2 | 2 | 1 | 2 | 2 | 9 | 🔴 丢弃 |
```

每项附简短评分理由。

### 阶段 5：苏格拉底深挖（深度发散）→ 对 Top-1 追问
**新增阶段**（来自"苏格拉底式追问融入 Wander Mode"，19分）：

对评分最高的那条 insight，追问 3 个 deepening questions：

```
## 🔍 深度追问：Top Insight

> **[最高分洞察原文]**

追问：
1. 这个结论的隐含前提是什么？如果前提不成立呢？
2. 有没有反例？什么情况下这个结论会失效？
3. 如果这个洞察是对的，下一步最值得探索的方向是什么？
```

这三个追问本身也会被评分，高分追问可以存为新的 open_question。

### 阶段 6：保存决策 + 用户确认
汇总保存建议，等待用户确认。

```
## ✅ 保存建议

🔵 长期记忆（≥18分）：X 条
🟡 临时缓冲（14-17分）：X 条
🔴 丢弃（<14分）：X 条

是否写入记忆？（y/n）
```

## 重要约束
- 这是**一次有限思考回合**，完成后自动停止
- **所有中间结果直接输出给用户**，不使用后台 agent
- 每阶段输出必须有**压缩摘要**（一句话），传递给下一阶段
- v0.1 用户确认后才写入记忆（除非 --no-save 跳过写入）
- 如果预筛选发现前 2 个种子联想质量都低，提前终止
- 总预算控制在 ~8000 token
