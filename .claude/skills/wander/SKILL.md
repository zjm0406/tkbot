# /wander — 空闲发散思考模式

触发智能体进行一次**有限发散思考回合**：提取种子 → 三层联想 → 价值筛选 → 输出保存建议。

## 参数
- `--seeds "词1,词2,词3"`：手动指定种子词，跳过自动提取
- `--no-save`：不输出保存建议，只展示思考结果

## 执行流程

### 第一步：准备
1. 读取 `memory/MEMORY.md` 和 `data/word_pool.json`
2. 告诉用户即将开始发散思考

### 第二步：提取种子
调用 `seed-extractor` 子代理，从上下文和记忆中提取 3-5 个种子。
（如果用户用了 `--seeds`，直接使用用户指定的种子）

### 第三步：发散联想
对每个种子调用 `wander-generator` 子代理，生成三层联想链。
多个种子可以并行处理。

### 第四步：价值筛选
将所有联想链和洞察交给 `value-filter` 子代理进行五维评分，
输出保存建议。

### 第五步：展示结果
用中文格式化展示：

```
🌿 Wander Mode — 本次发散思考结果

📋 种子词：xxx、xxx、xxx（来源：对话/记忆/随机）

🔗 联想链摘要：
  · xxx → xxx → xxx：一句话总结

💡 关键洞察：
  · xxx（评分：X分，建议：保存/缓冲/丢弃）

❓ 生成的问题：
  · xxx（评分：X分，建议：保存/缓冲/丢弃）

📊 保存建议汇总：
  建议长期保存：X 条
  建议临时缓冲：X 条
  建议丢弃：X 条
```

### 第六步：用户确认（默认）
向用户展示保存建议，询问是否写入记忆。
使用 `--no-save` 参数可跳过此步骤。

## 重要约束
- 这是**一次有限思考回合**，完成后自动停止
- **v0.1 不自动写入记忆**，只输出保存建议
- 如果种子提取或联想质量过低，提前结束并告知用户
- 整个过程控制在合理预算内，不要无限发散
