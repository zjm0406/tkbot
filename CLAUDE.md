# tkbot — Wander Mode 项目

## 项目定位
基于 Claude Code 的智能体模拟发散思考系统。
当前版本：v0.1（Claude Code 原型，手动触发）。

## 核心原则
- 生成时要发散，保存时要克制
- 一次触发 = 一次有限思考回合，自动停止
- v0.1 只输出保存建议，不自动写入记忆
- 不追求模拟人类意识，追求功能性认知行为

## 从 Wander 洞察中提取的架构原则

这些原则来自 v0.1 首次完整 Wander 的执行结果，被验证为有价值的架构指导：

### 原则 1：交替发散收敛（熵增-压缩平衡，24分）
**Wander Mode 的本质是发散（熵增）与收敛（压缩）的动态平衡。**
- 不要先全部发散再全部收敛——那会导致中间状态膨胀
- 正确模式：发散 → 压缩 → 再发散 → 再压缩
- 实现：阶段 2（联想扩散）之后立即做阶段 3（中间收敛），只有通过预筛选的链才进入深层评分

### 原则 2：问题价值 = 上下文 × 激发行动的概率乘数（24分）
**好问题的价值不是静态属性，而是它在特定上下文中激发后续有效行动的概率。**
- 一个洞察的价值可以通过它被后续 wander 引用、激发新想法的频率来反向度量
- 实现：记忆文件添加 `citation_count` 字段，被引用时递增
- 高引用次数的记忆在种子提取时获得更高权重

### 原则 3：扩展系统应设计"语法"而非预定义"名词"（22分）
**好的扩展系统是设计一套组合规则，让未知的用法能合法表达，而不是枚举所有可能。**
- Agent 定义应该写"联想规则"而非"输出模板"
- 实现：每个 agent 的核心是约束和原则，JSON schema 只是边界检查，不是思考框架
- 让模型有空间在规则内自由探索，而不是填模板

### 原则 4：涌现事后可解释但事前不可预测（22分）
**这正是 Wander Mode 存在的理由——如果所有结果都可预测，就不需要发散思考。**
- 不要试图在生成前就判断一个方向是否有价值
- 低约束生成 + 高约束筛选 = 涌现的土壤
- 实现：种子提取保留 1 个随机来源，联想第三层鼓励跨领域跳跃
- 每次 wander 记录完整元数据，用于事后分析哪些条件产生了好结果

### 原则 5：认知架构 = 信息压缩层次传递（21分）
**每层丢弃细节换取可操作性——压缩率决定了下层的效率。**
- 阶段间传递的不是原始 JSON，而是压缩后的"认知摘要"
- 实现：每个阶段输出必须包含一个一句话摘要，作为下一阶段的主要输入
- 如果某个阶段无法用一句话总结自己的发现，说明它自己也没想清楚

### 原则 6：苏格拉底追问 > 一次性联想（19分）
**能追问的机器比能联想的机器更有长期价值。**
- 联想产生候选洞察，追问深挖洞察的边界和前提
- 实现：评分最高的 insight 自动触发一轮 3 追问深挖
- 追问结果也可以存为 open_question，成为下次的种子

## 微思考协议（每次响应前执行）

**触发时机**：每次收到用户消息时，在回复前自动执行一次轻量级意图分析。
**设计来源**：用户提出的"每次发问都开启短思考"——这是 Wander Mode 的元应用。

### 三步微思考
每次响应用户前，快速执行：

1. **字面意图**：用户明确要求什么？（1 句话）
2. **隐含需求**：用户没说但暗示了什么？有什么未陈述的假设？
3. **长期价值**：这个交互中有值得存入记忆的非显而易见的洞察吗？

### 约束
- 总预算 ~200 token，不展开完整 Wander Mode
- 分析结果直接融入回复，不单独展示
- 只在发现 novelty ≥ 3 且 future_value ≥ 3 的洞察时，才写入 memory
- 如果用户偏好、习惯、思维模式被首次观察到 → 存入用户记忆

## 项目结构
- `.claude/agents/` — 子代理（seed-extractor, wander-generator, value-filter, socratic-probe, intent-analyzer）
- `.claude/skills/wander/` — /wander 命令
- `.claude/workflows/` — 多代理编排脚本
- `memory/` — 记忆存储
- `data/` — 静态数据（随机词库、源性能统计）
- `remind.md` — 完整设计文档

## 开发阶段
- v0.1：手动 /wander + 内联可见思考 + 保存建议（当前）
- v0.2：用户确认后写入记忆 + 引用追踪
- v0.3：远距组合增强 + 苏格拉底深挖
- v0.5：独立 runtime

## 技术约束
- v0.1 纯 Claude Code 原生机制，不写 Python
- 子代理间数据传递使用 JSON
- 不绑定单一模型后端
- 所有中间结果必须直接对用户可见，不使用后台 agent 隐藏
